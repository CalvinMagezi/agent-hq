How it works

From task to result in three steps

No cloud APIs in the critical path. No subscriptions. Your agents run locally, share context through the vault, and report results back to you.

01
📝
Describe a task

Type a task in Discord, the terminal chat, or create a job file directly. Agent-HQ's ambiguity detector classifies complexity and picks the right planning depth automatically.

→
02
🔄
Agents orchestrate

The HQ Agent picks up the job, delegates to the best harness (Claude Code, Gemini CLI, or OpenCode), generates diagrams, captures screenshots, and writes results back to the vault.

→
03
✅
Results in your vault

Every plan, every code change, every artifact lands in your Obsidian vault. Search it, link it, reference it across future sessions. The context compounds — each task makes the next one smarter.

Architecture

Vault-Centric Orchestration

Every agent reads from and writes to the same Obsidian vault. The vault is the shared brain — not a message queue, not a cloud API. Everything else plugs in around it.

agent

HQ Agent

Local worker that picks up jobs from the vault queue and executes them with Pi SDK tools — BashTool, FileTool, MCPBridgeTool.

relay

Discord Relay

Multi-bot system. Claude Code, Gemini CLI, and OpenCode plug in as equal harnesses for delegated tasks.

mcp

HQ Tools MCP

40+ tools behind a 2-tool gateway (hq_discover + hq_call). Fixed ~1K token footprint regardless of registry size.

cron

Daemon

Background workflows running on your machine: embeddings, heartbeat, knowledge extraction, plan maintenance, note linking.

Why local-first

Your data stays on your machine

Cloud AI tools make you dependent on SaaS subscriptions, rate limits, and remote servers that hold your context hostage. Agent-HQ is different.

Cloud AI tools

  • Context wiped between sessions — re-explain every time
  • Monthly subscription fees that compound across every tool
  • Your code and data leave your machine to remote servers
  • Rate limits throttle you when you need speed most
  • Vendor lock-in: switching models means rebuilding workflows
  • No continuity: one agent can't see what another did

Agent-HQ

  • Vault accumulates context across every session — no repeating yourself
  • Open source, MIT licensed, zero platform fees
  • All data stays in your Obsidian vault on your filesystem
  • No rate limits — local processes run at full speed
  • Swap harnesses (Claude, Gemini, OpenCode) without changing workflows
  • Shared vault means every agent inherits full project context

Get Started

Up in three commands

You need Bun v1.1.0+, an Obsidian vault path, and one API key.

01
Clone the repository
$ git clone https://github.com/CalvinMagezi/agent-hq && cd agent-hq
02
Install dependencies
$ bun install
03
Initialize Agent-HQ
$ bun run hq init

No. Agent-HQ uses Obsidian's vault format (plain markdown files in a folder) but doesn't require the Obsidian app to be installed or running. You can use any text editor to view vault files. If you do use Obsidian, the free version is sufficient.
Agent-HQ routes through OpenRouter by default, giving you access to Claude, Gemini, GPT-4, Llama, and more. Gemini models also route directly via the Gemini API when GEMINI_API_KEY is set. The harness system (Claude Code, Gemini CLI, OpenCode) is separate from the orchestration model.
The core vault, daemon, and HQ tools work on any platform where Bun runs (macOS, Linux, Windows WSL). The hq install command sets up macOS launchd for background services. Linux systemd support and Windows native are planned.
AutoGen and CrewAI are Python frameworks for defining agent graphs in code. Agent-HQ is a vault-centric operating environment — agents are existing CLI tools (Claude Code, Gemini CLI) that plug in as harnesses. The vault accumulates persistent context across all sessions. There's no Python, no agent graph definition, and no ephemeral in-process state.
Yes. Discord is one interface, not a requirement. You can create jobs via the terminal chat (bun run chat), via direct vault file creation, or via the MCP gateway from Claude Code. The Discord relay is an optional component.
